# Implementation Strategy

## Content

Retained for documentation continuity.

damspy-core is an existing working repository and does not use this document as a feature-by-feature planning log.

This file records only the broad implementation posture of the repository:

- preserve working measurement behaviour
- prefer incremental change over redesign
- keep hardware and external service integrations explicit and contained
- avoid architectural speculation that is not justified by real repo needs
- if documentation conflicts with working code, treat the working code as current truth and update the docs accordingly

For current repo guidance, use:

- `README.md`
- `doc_map.md`
